2.1.1 Predicting Rejected Loans¶

Our first model is used to predict whether a loan application was accepted or rejected. It uses a table that was merged from the accepted and rejected loan (only the independent variables that are available in both are included).

Accepted/Rejected is encoded as 0/1 in our traing/test samples

2.1.1.1 Model Selection and Hypertuning¶

We've experimented using multiple models:

  • XGBoost
  • LGBM
  • CatBoost

And multiple tuning target metrics:

  • F1
  • PR-AUC
  • LogLoss

When predicting rejected application we have achieved the best performance when using an XGBoost (as fast or faster than CatBoost and LGBM with slightly better performance) tuning to maximize F1.

see the appendices for more details.

2.1.1.2 Training and Evaluating the Accepted/Rejected Application Model¶

The dataset is pre-split into a 20% test sample and a 80% training sample.

Overall our goal was to maximize the macro F1 (we have no reasons to priortize the accuracy for either class because the model doesn't really have any practical applications besides estimating the algorithm Lenders Club is using to reject applications and any potential biases that it might contain).

Additionally, we've also used PR-AUC to evaluate our model, it quantifies a classifier's ability to distinguish between classes under varying thresholds (however this will be more relevant for the default risk estimation model).

We have been able to achieve very solid results both when tuning for F1 and PR-AUC with F1=~0.9 and PR-AUC =~0.98.

The model performs significantly better at predicting rejected loans (~17% of accepted loans are misclassified). Depending on how the model would be used we might consider using FBeta, non-macro F1 or even accuracy as our tuning target to maximize the performance for a specific class.

We can see that our model is signficantly better at predicting positive (i.e. loan accepted) than negative cases, with quite a few rejected cases being misclassified as accepted. We could potentially use different hyper-parameters to optimize for precision instead of overal predictive power if our business problem required that (we'll see some example of that when building the Default Risk model in part 2.2
